# CryptoTW Pro Mobile Card 微調與空間優化指南

本指南旨在解決 Mobile UI 卡片「左重右輕、空間浪費」的問題，建立一致且平衡的視覺語言。

## ① 卡片左右空間失衡的「常見原因清單」

1. **過度依賴垂直堆疊 (Flex-col Overuse)**
    * 開發時習慣將所有資訊（標題、副標、數值、標籤）全部放進一個左對齊的 `flex-col` 容器中，導致左側擁擠，右側全空。
2. **右側缺乏視覺錨點 (Missing Right Anchor)**
    * 右側往往只有一個箭頭 (`ChevronRight`) 或完全留白，無法與左側的重內容形成視覺抗衡。
3. **文字資訊流無分流 (Unbalanced Information Flow)**
    * 日期、時間、狀態等「輔助資訊」被混入左側主標題下方，造成閱讀視線必須垂直掃描，而不是自然的 Z 字型或水平掃描。
4. **Padding 視覺體感不一 (Inconsistent Visual Padding)**
    * 雖然 CSS padding 數值可能一樣，但因為內容密度不同（左密右疏），導致使用者體感上覺得「左邊很擠，右邊很空」。
5. **缺乏橫向節奏 (Lack of Horizontal Rhythm)**
    * 在列表型卡片中，沒有固定的右側欄位（如時間或狀態），導致每行長度不一，視覺顯得凌亂。

## ② Mobile 卡片空間微調原則

1. **非線性佈局 (Non-linear Layout)**
    * 拒絕從上到下一條線。內容必須「左右分工」，左邊放主體（What），右邊放情境（When/How/Status）。
2. **右側強制錨點 (Mandatory Right Anchor)**
    * 每張卡片的右側（右上或右下）必須有一個視覺元素（Icon、Badge、數據、圖表）來「壓住」版面，平衡左側重量。
3. **對角線平衡 (Diagonal Balance)**
    * 如果你在「左上」放了重標題，請在「右下」或「右上」放一個強調色或圖形，形成對角線平衡。
4. **輔助資訊右置 (Secondary Info to the Right)**
    * 時間 (`10:00`)、狀態 (`Bullish`)、變動率 (`+5%`) 應優先考慮放在右側，而不是堆在標題下面。
5. **膠囊化數據 (Capsule Metrics)**
    * 底部數據區塊盡量使用水平排列的「膠囊」樣式，增加橫向視覺延伸感。

## ③ 通用卡片結構建議

所有卡片應盡量符合以下抽象結構：

```
+-------------------------------------------------------+
|  [Header 區]                                          |
|  左：主標題 / 幣種 / 名稱 (粗體, 高對比)              |
|  右：狀態標籤 / 影響力等級 / 重點 Tag (視覺錨點)      |
+-------------------------------------------------------+
|  [Body 內容區]                                        |
|  左：核心數值 / 結論文字 (大字號或多行)               |
|  右：輔助圖表 / 趨勢線 / 次要數據 (避免留白)          |
+-------------------------------------------------------+
|  [Footer / Meta 區] (可選)                            |
|  橫向排列： 時間 • 來源 • 相關性 (灰色資訊)           |
|  或是：數據膠囊 A | 數據膠囊 B (水平撐開)             |
+-------------------------------------------------------+
```

## ④ 各類卡片的微調方向

### 1. 主卡 (Primary Card - e.g., DecisionHero)

* **現狀**：左側資訊尚可，但底部 Grid 可能過於密集。
* **微調方向**：
  * **Header**：保持 BTC/USDT 在左，將「多空狀態 Badge」明確固定在右上角（目前已有，需確認視覺重量）。
  * **Body**：價格與漲跌幅若堆疊在左，可考慮將「漲跌幅」移至價格右側或右方，讓數值橫向展開。
  * **Bottom**：健康指標若為 Grid，確保每個 Item 充滿寬度，或改為橫向滑動/膠囊列，避免看起來像是一堆小方塊擠在左下。

### 2. 列表卡 (List Card - e.g., UpcomingEvents, News)

* **現狀**：左側標題+倒數，視覺容易混亂。
* **微調方向**：
  * **左側 (Flex-1)**：事件名稱 (Truncate 1-2行)。
  * **右側 (Fixed Width)**：建立一個固定寬度的右欄（例如 `w-[80px]`），專門放置「倒數時間」或「勝率」。
  * **分離**：不要讓「倒數時間」跟在標題後面，把它強制拉到右側，形成明確的 `標題 ------- 時間` 結構。

### 3. 指標卡 (Indicator Card - e.g., SentimentMatrix)

* **現狀**：典型左重右輕。標題、數值、狀態都在左邊，右邊全空。
* **微調方向**：
  * **對角線佈局**：
    * 左上：指標名稱。
    * 左下：大數值 (Value)。
    * **右上/右下**：將「狀態 Badge」(e.g., 貪婪) 移至 **右上角** 或 **右下角**。
  * 這樣卡片的四個角落至少有三個有內容，視覺重心就會平衡。

## ⑤ Mobile Card Balance Checklist

在提交設計前，請檢查：

* [ ] **左右平衡**：是否每張卡片的右半部都有至少一個視覺元素（文字除外）？
* [ ] **視覺錨點**：右上角是否有狀態、Icon 或數據？
* [ ] **非線性閱讀**：是否成功避免了「所有資訊都在左側垂直堆疊」？
* [ ] **邊距一致**：左右 Padding 是否視覺上看起來相等？
* [ ] **空間利用**：右側是否消除了無意義的留白（刻意留白除外）？
* [ ] **操作暗示**：如果是可點擊卡片，右側是否暗示了可互動性（透過佈局而非僅靠箭頭）？
